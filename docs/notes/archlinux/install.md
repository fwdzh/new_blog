其实我也不知道为啥要安装 Archlinux ，但是用了一段时间之后发现挺好用的。比起之前用过的 Ubuntu 流畅很多，大于等于 Windows 大概。

比较难受的就是，linux 上的办公软件我感觉不是很好用。用得比较多的是 libreoffice 和 wps，我在 windows 上只用过 microsoft office，这个应该是最好用的吧。

虽然现在很多游戏在 linux 上也能玩，但是也很多不能玩。steam 上有的游戏实际上是通过 wine 来运行的，性能会有一些损失。有些游戏说是能在 linux 运行，但我安了之后显示比例很奇怪。
:::

安装其实就是挂载一个根分区，一个 EFI 分区，根分区就是 `/` 目录所在的地方，EFI 分区用来放一些启动信息之类的。swap 分区不要也行，而且现在可以直接用 swap 文件不需要分区，可以安装好后再去加。而且感觉这玩意大部分时候都用不上。

进了 Live 安装环境后，我们可以使用 cfdisk 等工具划分空间，我感觉我自己安装 archlinux 花的时间最多的可能就是分区了，虽然一般只需要分一个根分区。

## EFI 分区空间不足

如果电脑上已经有 Windows，那大概率是已经有 EFI 分区的，但是可能空间不够……我装的时候，一直提示 EFI 空间不足。第一次遇到这个时，我新开了一个 EFI 分区，并且删除了原有的 EFI 分区，于是安装成功后，我只能进入 Archlinux。制作 Windows 安装盘之后，可以修复 EFI 分区，然后不能启动 Arch，再进 Live CD 重新挂载 EFI 就好了。

第二次遇到这个问题时，还是不知道怎么扩充 EFI 分区，因为释放后面的空间的话，只能扩充后面的空间，但是 EFI 分区一般是在最前面。我 EFI 分区后面空了 16 MB 的空间，把它扩充到 EFI 之后，free -h 查看的空间还是没变，于是复制一份，格式化，再复制回来，就启动不了了。不知道中间是哪里出了问题，后面也懒得去研究了，直接把磁盘擦除重新安装!

所以如果 EFI 分区空间足够，想清楚分区怎么分，安装会挺快的，就只看网络速度了……然后擦除磁盘安装，很简单，哈哈哈，感觉挺爽的，但应该很多人还是需要一个 Windows。

然后没有 dock 栏，像 Windows 那种，底下显示已打开的应用，Ubuntu 也可以。
